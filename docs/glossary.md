# Cybersecurity Glossary

My personal collection of important cybersecurity terms and definitions.

**Last Updated**: July 02, 2026  
**Total Terms**: 42

## A

**Attack Surface**  
All possible points where an attacker can try to enter or breach a system.

**Attack Vector**  
The method or path used by an attacker to breach a system (e.g., phishing, malware, vulnerability exploit).

**Authentication**  
The process of verifying the identity of a user, device, or system.

**Authorization**  
The process of determining what an authenticated user is allowed to do.

## B

**Blue Team**  
The defensive security team focused on detection, monitoring, and protection.

**Brute Force Attack**  
Trying many possible passwords or keys until the correct one is found.

**Bug Bounty**  
A program where ethical hackers are rewarded for finding and reporting vulnerabilities.

## C

**CIA Triad**  
The three core principles of cybersecurity:  
- **Confidentiality** – Protecting information from unauthorized access  
- **Integrity** – Ensuring information is accurate and not tampered with  
- **Availability** – Ensuring information and systems are accessible when needed

**Command and Control (C2)**  
Infrastructure attackers use to communicate with and control compromised systems.

**Cryptography**  
The practice of securing information through encryption and decryption.

## D

**Defense-in-Depth**  
A security strategy that uses multiple layers of protection so that if one fails, others still provide protection.

**Denial of Service (DoS)**  
An attack that makes a system or network unavailable to legitimate users.

**Digital Signature**  
A cryptographic technique used to verify the authenticity and integrity of a message or document.

## E

**Encryption**  
The process of converting readable data into unreadable format to protect it.

**Exploit**  
Code or technique that takes advantage of a vulnerability in a system.

## F

**Firewall**  
A security system that monitors and controls incoming and outgoing network traffic.

## H

**Hashing**  
Converting data into a fixed-length string (used for passwords and integrity checks).

## I

**Incident Response**  
The process of detecting, responding to, and recovering from a cybersecurity incident.

**Intrusion Detection System (IDS)**  
A system that monitors network traffic for suspicious activity.

**Intrusion Prevention System (IPS)**  
An IDS that can also actively block attacks.

## L

**Least Privilege**  
Principle that users and applications should have only the minimum permissions needed.

## M

**Malware**  
Malicious software designed to harm or exploit systems (viruses, trojans, ransomware, etc.).

**Man-in-the-Middle (MitM)**  
An attack where the attacker secretly intercepts communication between two parties.

## O

**OWASP**  
Open Web Application Security Project — an organization that publishes security standards.

## P

**Patch**  
A piece of software designed to fix vulnerabilities or bugs.

**Payload**  
The part of malware that carries out the malicious action.

**Penetration Testing (Pentesting)**  
Simulated cyber attack to find vulnerabilities in a system.
